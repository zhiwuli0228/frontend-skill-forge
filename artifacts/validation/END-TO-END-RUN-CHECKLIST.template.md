# End-to-End Run Checklist

Use this checklist before closing any benchmark-grade Skill + MCP + cross-module validation run.

## Preconditions

- [ ] run id created
- [ ] `manifest.md` created under `artifacts/validation/runs/<run-id>/`
- [ ] target Skill identified
- [ ] initial minimal context recorded
- [ ] MCP availability checked

## Skill Layer

- [ ] primary Skill named
- [ ] Skill-only planning recorded
- [ ] deliberate non-loaded context recorded
- [ ] any escalation recorded

## MCP Layer

- [ ] MCP-first exploration attempted
- [ ] MCP session record created or fallback reason recorded
- [ ] initial visual evidence captured
- [ ] interaction trace captured

## Change Layer

- [ ] affected modules listed
- [ ] affected routes listed
- [ ] affected source files listed
- [ ] cross-module boundary justification recorded
- [ ] change remains bounded

## Runtime Evidence Layer

- [ ] at least 2 screenshots saved
- [ ] HTML report or trace saved
- [ ] raw logs saved
- [ ] machine-readable summary saved
- [ ] pass / partial / fail / blocked conclusion written

## Self-Evolution Layer

- [ ] anomalies recorded
- [ ] repeated patterns distinguished from one-off noise
- [ ] candidate target identified: Skill / rule / knowledge / none
- [ ] queue update made or explicit no-candidate conclusion recorded

## Closeout

- [ ] docs point to raw artifact paths
- [ ] context behavior conclusion written
- [ ] next-step recommendation written
