## Hierarchy Policy

- Major versions are the primary release units and require your acceptance before closure or progression.
- Major versions may contain multiple minor versions or gated sub-iterations.
- Minor versions are used to stage work inside a major version without falsely claiming the major version is complete.
- A major version is not complete until its final aggregate summary and user acceptance are both recorded.

## Scope Boundary

In scope:

- Versioned governance baselines and gate records
- OpenSpec, SuperSpec, and Superpower collaboration design
- Static spec and schema assets required for the collaboration framework
- Evidence-backed implementation, review, and verification
- Agent-facing knowledge capture and decision traceability
- Accepted V1 release records and retrospective outputs
- First code-bearing frontend runtime surfaces for evidence-driven experimentation

Out of scope:

- Untracked ad hoc implementation work
- Business/frontend implementation before governance activation is accepted
- Closing a version without linked verification evidence
- Scattering change history across unrelated docs
- Treating placeholder pages as accepted validation baselines
- Real backend integration or production-grade business completeness inside V2

## Governance Rules

- Current status is updated only here, not duplicated across multiple READMEs.
- User confirmations and direction changes are appended only to `current/DECISIONS.md`.
- Main design and implementation outputs are versioned under `baselines/Vx/`.
- Each version keeps exactly one release or blocked-state summary in `releases/Vx.md`.
- Aggregate summaries, issues, and optimizations may be added under their dedicated directories when a major version closes.
- Temporary notes, review scratchpads, and duplicated progress summaries should not remain as permanent HEAD documents unless promoted into a baseline or release record.
- Major-version progression requires explicit user acceptance.
- Major-version closure requires a final aggregate summary across its minor versions.
