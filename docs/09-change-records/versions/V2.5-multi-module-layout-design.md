# V2.5 Design: Multi-Module Complex Layout

## 1. Background

### 1.1 Current State (V2.4)

V2 系列已完成三个运行时基线变更：
- Dashboard 运行时基线（V2.2）
- Task List 运行时基线（V2.3）
- Task Create 运行时基线（V2.4）

当前布局结构为简单的 Sider + Content 二栏布局，仅支持 Dashboard 和 Task 两个业务域。作为复杂页面自动化工具探索项目，当前页面复杂度不足以覆盖多样化的前端交互模式。

### 1.2 Motivation

提升页面复杂度以覆盖更多前端自动化场景：
- 多级导航结构（模块切换 + 模块内导航）
- 多种内容展示模式（表格、卡片网格、可视化画布、图表）
- 复杂交互模式（拖拽、多步表单、视图切换、抽屉/模态框联动）
- 场景驱动 UI 在多模块下的统一实践

## 2. Target Architecture

### 2.1 Layout Structure

```
┌─[☰]── Logo ──────────────────────────────────────────────┐
│  Header: 模块切换按钮 + 当前模块名 + 搜索 + 用户头像       │
├───────────────────────────────────────────────────────────┤
│  [SubTab1] [SubTab2] [SubTab3] [SubTab4]                 │
├──────────┬──────────────────────────────────────────────┤
│          │  面包屑                                       │
│  侧边    │──────────────────────────────────────────────│
│  菜单    │                                              │
│          │       主内容区                                │
│  · 子页1  │       (表格/卡片/画布/图表...)                │
│  · 子页2  │                                              │
│  · 子页3  │                                              │
└──────────┴──────────────────────────────────────────────┘
```

### 2.2 Component Hierarchy

```
App
└── Providers
    └── ConfigProvider
        └── RouterProvider
            └── GlobalShell              ← 全局壳（Header + 模块切换抽屉）
                ├── ModuleLayout(Task)    ← 任务中心模块
                │   ├── ModuleTabs
                │   ├── ModuleSidebar
                │   └── ModuleContent → TaskListPage / TaskCreatePage / ...
                ├── ModuleLayout(Skill)   ← 技能库模块
                ├── ModuleLayout(Workflow) ← 工作流模块
                ├── ModuleLayout(Insight)  ← 数据洞察模块
                └── ModuleLayout(Settings) ← 系统设置模块
```

## 3. Module Specifications

### 3.1 Task Center (任务中心)

**Purpose**: 任务管理全流程，覆盖表格、表单、看板等核心交互模式。

**Tabs**: 任务列表 | 创建任务 | 模板库 | 看板视图

**Sidebar**: 筛选面板
- 状态筛选：全部 / 进行中 / 已完成 / 已归档
- 优先级筛选：全部 / 高 / 中 / 低
- 分类筛选：全部 / 开发 / 设计 / 测试 / 运维

**Pages & Components**:

| Page | Key Components | Interaction Patterns |
|------|---------------|---------------------|
| TaskListPage | TaskFilterBar, TaskTable, TaskDetailDrawer | 复杂表格（筛选+排序+批量+分页）、行点击→抽屉详情 |
| TaskCreatePage | TaskCreateForm, TaskPreview | 多步表单（3步）、实时预览、表单验证 |
| TaskTemplatesPage | TemplateCard, TemplateCategory | 卡片网格、分类筛选、模态框预览 |
| TaskBoardPage | BoardColumn, TaskCard | 看板拖拽、列管理、卡片快速编辑 |

**Scenario Selector**: loaded | loading | empty | error | validation

### 3.2 Skill Library (技能库)

**Purpose**: 技能配置与管理，覆盖卡片网格、视图切换、代码编辑器等模式。

**Tabs**: 技能列表 | 技能市场 | 配置中心 | 版本管理

**Sidebar**: 分类导航
- 全部技能
- 文本处理
- 图像生成
- 数据分析
- 工具调用

**Pages & Components**:

| Page | Key Components | Interaction Patterns |
|------|---------------|---------------------|
| SkillListPage | SkillCard, ViewToggle, SkillFilterBar | 卡片/列表视图切换、搜索、标签筛选 |
| SkillMarketPage | MarketCard, CategoryNav, InstallButton | 市场浏览、一键安装、详情模态框 |
| SkillConfigPage | ConfigEditor, ParamForm, TestRunner | 代码编辑器、参数表单、测试执行 |
| SkillVersionsPage | VersionTimeline, DiffViewer | 版本时间线、差异对比、回滚操作 |

**Scenario Selector**: loaded | loading | empty | error

**Tabs**: 工作流列表 | 编排器 | 执行历史 | 调度管理

**Sidebar (编排器页面)**: 节点库

- 判断节点
- 输出节点

**Pages & Components**:

|------|---------------|---------------------|
| WorkflowListPage | WorkflowTable, StatusTag, ActionMenu | 表格操作、状态标签、批量管理 |
| WorkflowEditorPage | Canvas, NodePanel, PropertyDrawer | 画布拖拽、节点连线、属性编辑 |
| WorkflowHistoryPage | ExecutionTable, LogViewer | 执行日志表格、日志详情查看 |
| WorkflowSchedulePage | CronEditor, ScheduleList | Cron 表达式编辑、调度列表管理 |

**Scenario Selector**: loaded | loading | empty | error

### 3.4 Data Insight (数据洞察)

**Purpose**: 数据可视化与分析，覆盖多种图表、数据表格、筛选联动等模式。

## 4. Shared Components (新增)

### 4.1 GlobalShell

文件: `src/shell/layout/GlobalShell.tsx`

职责:
- 渲染 Header（Logo + ☰ 按钮 + 模块名 + 搜索 + 用户头像）
- 管理模块切换抽屉（Drawer 展示 5 个模块卡片）
- 渲染当前模块的 `<Outlet />`

### 4.2 ModuleLayout

文件: `src/shell/layout/ModuleLayout.tsx`

职责:
- 接收 `tabs`、`sidebarMenu`、`title` 等 props
- 渲染模块内 Tabs 栏
- 渲染模块内 Sider + Menu
- 渲染 Content + Breadcrumb + `<Outlet />`

### 4.3 ModuleSwitcher

文件: `src/shell/navigation/ModuleSwitcher.tsx`

职责:
- 渲染 Drawer 内的模块卡片网格
- 点击卡片导航到对应模块
- 高亮当前模块

### 4.4 TopTabNavigation

文件: `src/shell/navigation/TopTabNavigation.tsx`

职责:
- 渲染模块内 Tabs
- Tab 切换时更新路由
- 保持 Tab 状态

## 5. Compliance with Project Conventions

### 5.1 Domain Structure

每个新模块遵循现有 domain 结构：

```
src/domains/<module>/
  pages/           ← Page 组件（路由目标）
  components/      ← 模块内复用组件
  data/            ← Mock 数据 + 类型定义
```

### 5.2 Component Patterns

- Function 组件 + named export
- 独立 TypeScript interface 定义 props
- 本地 useState 管理状态
- 场景选择器（loaded/loading/empty/error）
- data-testid 属性覆盖关键元素

### 5.3 Data Patterns

- Mock 数据静态定义在 `data/mock-data.ts`
- 筛选常量使用 `as const` 断言导出
- 空状态数据单独导出（如 `emptyTasks`）
- 不引入 API 调用或异步数据加载

### 5.4 Ant Design Usage

- 使用 Ant Design 6 组件：Layout, Menu, Tabs, Drawer, Card, Table, Form, Tree, Steps, Modal, Breadcrumb, etc.
- 使用 @ant-design/icons 图标
- 内联样式布局
- ConfigProvider 主题包裹
