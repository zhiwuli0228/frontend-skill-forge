# V2.1 Dispatch Contract

Status: Draft baseline

Version: V2.1-DISPATCH-CONTRACT

Date: 2026-06-07

## Purpose

Define the bounded dispatch format for any agent assigned to V2.1 version-design work.

## Required Dispatch Fields

- target branch
- target minor version
- target stage
- allowed files
- forbidden files
- required reads
- validation commands
- evidence output path
- final report format

## V2.1-Specific Boundaries

Agents working in V2.1 must not:

- implement runtime page code yet
- claim that dashboard, task list, or task creation are already complete
- reduce V2 to a trivial CRUD or placeholder refresh
- introduce backend dependencies

## Required Output Types

- version-design documents
- page-architecture baselines
- gate and dispatch contracts

## Acceptance Rule

Any V2.1 output that weakens the complexity or testability goals of V2 without a recorded user decision must be rejected.
