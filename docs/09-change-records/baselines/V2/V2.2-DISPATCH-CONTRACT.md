# V2.2 Dispatch Contract

Status: Draft baseline

Version: V2.2-DISPATCH-CONTRACT

Date: 2026-06-07

## Purpose

Define the bounded dispatch format for any agent assigned to V2.2 dashboard implementation work.

## Required Dispatch Fields

- target branch
- target minor version
- target stage
- allowed files
- forbidden files
- required reads
- validation commands
- evidence output path
- final report format

## V2.2-Specific Boundaries

Agents working in V2.2 must not:

- implement task-list or task-create runtime scope except for narrowly justified shared primitives
- introduce backend integration
- leave the dashboard as a cosmetic-only placeholder replacement
- weaken selector stability or evidence-oriented state visibility

## Required Output Types

- dashboard implementation files
- dashboard verification updates
- dashboard gate records

## Acceptance Rule

Any V2.2 output that produces a prettier dashboard without meaningful statefulness, nested interaction, and testability must be rejected.
