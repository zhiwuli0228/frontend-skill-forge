# V1.2 Dispatch Contract

Status: Closed for V1.2

Version: V1.2-DISPATCH-CONTRACT

Date: 2026-06-07

## Purpose

Define the bounded dispatch format for any agent assigned to V1.2 documentation work.

## Required Dispatch Fields

- target branch
- target minor version
- target stage
- allowed files
- forbidden files
- required reads
- validation commands
- evidence output path
- final report format

## V1.2-Specific Boundaries

Agents working in V1.2 must not:

- change `src/`
- change `src/`
- change `tests/`
- change business/runtime implementation outside static schema assets
- claim that `superspec` is active
- create Superpower implementation files

## Required Output Types

- baseline design documents
- review/disposition/closure records
- acceptance-precheck records

## Acceptance Rule

Any agent output that implies runtime activation without implementation evidence must be rejected.
