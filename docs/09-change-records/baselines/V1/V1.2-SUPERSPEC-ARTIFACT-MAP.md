# V1.2 SuperSpec Artifact Map

Status: Closed for V1.2

Version: V1.2-MAP

Date: 2026-06-07

## Purpose

Map the intended `superspec` schema outputs to repository locations and gate responsibilities.

## Artifact Map

| Schema Artifact | Purpose | Repository Target |
| --- | --- | --- |
| brainstorm template | Pre-change exploration output | `docs/03-openspec/superpower/` |
| proposal template | Scope and delivery baseline | `openspec/changes/<change>/proposal.md` |
| specs template | Functional and E2E requirement baseline | `openspec/changes/<change>/specs/**/*.md` |
| design template | Technical design baseline | `openspec/changes/<change>/design.md` |
| plan template | Skill, MCP, and evidence execution plan | `openspec/changes/<change>/plan.md` |
| tasks template | Trackable implementation and verification checklist | `openspec/changes/<change>/tasks.md` |
| implementation template | Execution record | `openspec/changes/<change>/implementation.md` |
| verify template | Executed verification and evidence record | `openspec/changes/<change>/verify.md` |
| review template | Independent review output | `docs/09-change-records/baselines/Vx/*-REVIEW.md` |
| disposition template | Resolution record | `docs/09-change-records/baselines/Vx/*-DISPOSITION.md` |
| closure template | Closure verification record | `docs/09-change-records/baselines/Vx/*-CLOSURE.md` |
| acceptance-precheck template | Reverse traceability check | `docs/09-change-records/baselines/Vx/*-ACCEPTANCE-PRECHECK.md` |
| release template | Major-version summary | `docs/09-change-records/releases/Vx.md` |

## Ownership Map

| Artifact Type | Primary Owner | Control |
| --- | --- | --- |
| change shell | OpenSpec | command lifecycle |
| workflow stages | SuperSpec | schema contract |
| brainstorm inputs | Superpower | skill governance |
| dispatch/governance inputs | Superpower | mission/power contract |
| version status | Version ledger | current-state authority |

## Constraint

This map is normative for future implementation work. It is not proof that the schema already generates these artifacts.
