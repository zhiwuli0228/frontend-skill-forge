# V1.4 Dispatch Contract

Status: Closed for V1.4

Version: V1.4-DISPATCH-CONTRACT

Date: 2026-06-07

## Purpose

Define the bounded dispatch format for any agent assigned to V1.4 integration-verification work.

## Required Dispatch Fields

- target branch
- target minor version
- target stage
- target mission
- allowed files
- forbidden files
- required reads
- runtime validation commands
- evidence output path
- final report format

## V1.4-Specific Boundaries

Agents working in V1.4 must not:

- change `src/`
- change `tests/`
- create fake implementation evidence
- claim V1 major-version closure without explicit user acceptance
- convert control-plane proof into business/runtime approval

## Required Output Types

- integrated walkthrough record
- traceability map
- gate records
- acceptance-precheck records

## Acceptance Rule

Any V1.4 output that reports integration without command-backed evidence, or that collapses minor-version readiness into major-version acceptance, must be rejected.
