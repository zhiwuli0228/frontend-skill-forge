Status: Closed for V1.4

Version: V1.4-GATE-CONTRACT

## Purpose

Define the V1.4 gate sequence for integration and gate verification.

## V1.4 Scope

V1.4 covers:

- integrated walkthrough across OpenSpec, SuperSpec, and Superpower
- traceability from V1 requirements to gate artifacts
- V1 major-version acceptance precheck material
- blocked-item visibility if any gap remains

V1.4 does not cover:

- business/frontend implementation
- automatic closure of V1 without user acceptance
- claiming that future code-bearing changes no longer need verification

## Required Gates

1. Design review
2. Design disposition
3. Design closure verification
4. Implementation review
5. Implementation disposition
6. Implementation closure verification
7. Acceptance precheck
8. Acceptance precheck verification

## Exit Condition

V1.4 is ready for user acceptance review when:

- the integrated walkthrough exists
- the traceability map exists
- the V1.4 gate records are complete and not blocked
- command-backed evidence supports the main control-plane claims
- no V1.4 output implies that V1 major acceptance has already happened
