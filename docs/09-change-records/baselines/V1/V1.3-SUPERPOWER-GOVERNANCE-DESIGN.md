# V1.3 Superpower Governance Design Baseline

Status: Closed for V1.3

Version: V1.3-DESIGN

Date: 2026-06-07

## Purpose

Define the Superpower governance layer that controls brainstorming, mission dispatch, claim discipline, and review separation for V1.

## Problem

V1.2 established the SuperSpec schema baseline, but the repository still lacked a governed way to:

- distinguish brainstorming from approved execution
- dispatch bounded work to agents
- control what agents may claim about completion or activation
- keep review, disposition, and closure independent

Without this layer, OpenSpec and SuperSpec would exist, but agent execution would remain partially ad hoc.

## Design Goals

V1.3 must:

1. define mission types for exploration, design, implementation, review, and closure
2. define powers that constrain claims and scope expansion
3. establish explicit brainstorming rules
4. establish dispatch and review governance compatible with the V1 gate model
5. preserve the no-business-code boundary from `D015`

## Core Design Decisions

### Decision 1: Superpower governs how agents work, not what OpenSpec stages exist

OpenSpec remains the change engine and SuperSpec remains the workflow schema. Superpower adds a control layer over agent behavior, especially before and between formal stage artifacts.

### Decision 2: Brainstorming is a governed pre-change mode

Brainstorming is allowed as a discovery action, but it cannot imply approval, activation, or verification. It must terminate in a controlled handoff.

### Decision 3: Missions and powers are separate concepts

Missions describe the kind of work being done. Powers describe the authority and guardrails that apply while doing it. This avoids mixing workflow intent with governance controls.

### Decision 4: Review separation is mandatory

V1.3 adopts the same gate philosophy as the rest of V1: review, disposition, and closure are separate checkpoints and may not be collapsed into one undocumented success claim.

## Explicit Non-Goals

V1.3 does not:

- prove end-to-end integration across OpenSpec, SuperSpec, and Superpower
- authorize runtime code changes under `src/` or `tests/`
- replace V1.4 integration and gate verification

## Conclusion

V1.3 completes the missing governance layer between schema definition and full integration. After this baseline, the repository has documented controls for brainstorming and agent execution rather than relying on informal operator judgment.
