## Rules

### 1. Feedback Types

- **Correction:** "This is wrong"
- **Enhancement:** "This could be better"
- **Confirmation:** "This is correct"
- **Rejection:** "Don't do this"

### 2. Recording Requirements

Every feedback must include:
- Timestamp
- Source (human identifier)
- Feedback text
- Context (what was being done)
- Action taken

### 3. Feedback Processing

1. Record feedback
2. Assess validity
3. Create evolution candidate if valid
4. Track in evolution queue
5. Report resolution

### 4. Feedback Prioritization

- **Critical:** Blocks functionality
- **High:** Significant improvement
- **Medium:** Minor improvement
- **Low:** Nice to have
